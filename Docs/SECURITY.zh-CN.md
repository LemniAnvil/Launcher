## 报告漏洞

我们非常重视 Minecraft Launcher 的安全性。如果您发现安全漏洞,请通过负责任的方式报告,帮助我们保护用户。

### 如何报告

**请勿通过公开的 GitHub Issues 报告安全漏洞。**

请通过以下方式报告安全漏洞:

1. **在 GitHub 上开启私密安全公告**:
   - 访问 [Security Advisories](https://github.com/LemniAnvil/Launcher/security/advisories) 页面
   - 点击 "Report a vulnerability"
   - 填写漏洞详细信息

2. **直接联系维护者**:
   - 开启私密 issue(如果可用)
   - 通过项目中提供的联系方式联系

### 应包含的信息

报告漏洞时,请包含:

- **描述**: 清晰描述漏洞
- **影响**: 问题的潜在影响和严重程度
- **重现步骤**: 详细的重现步骤
- **概念验证**: 如果可能,包含概念验证或漏洞利用代码
- **受影响版本**: 哪些版本的启动器受到影响
- **修复建议**: 如果您有修复建议

### 响应时间

- **初始响应**: 我们力求在 48 小时内确认收到您的漏洞报告
- **状态更新**: 我们将定期提供漏洞处理进度的更新
- **解决方案**: 我们将尽快解决关键漏洞,通常在 7-14 天内解决关键问题

### 披露政策

- 我们遵循**协调披露**方法
- 我们请求您在我们发布修复之前不要公开披露漏洞
- 修复发布后,我们将在安全公告和发布说明中署名(如果您愿意)
- 我们力求在初始报告后 90 天内发布安全修复

## 安全注意事项

### 账户安全

本启动器支持 Microsoft 账户认证。请注意:

- **切勿与任何人分享您的账户凭据**
- **为您的 Microsoft 账户使用强密码和唯一密码**
- **在您的 Microsoft 账户上启用双因素认证**
- 启动器**从不存储您的密码** - 仅存储认证令牌
- 认证令牌安全地存储在 macOS 钥匙串中

### 下载安全

启动器从官方 Mojang/Microsoft 服务器下载游戏文件:

- 所有下载都使用 **SHA1 校验和**验证
- 文件通过 **HTTPS** 连接下载
- 启动器在安装前验证文件完整性
- 仅使用官方 Mojang/Microsoft 服务器进行游戏下载

### 代理安全

如果您使用代理设置:

- 使用第三方代理时要谨慎
- 仅使用来自可信来源的代理
- 代理凭据(如果有)安全存储
- SOCKS5 代理比 HTTP 代理提供更好的安全性

### 文件系统安全

启动器在标准 Minecraft 目录内运行:

- 默认位置: `~/.minecraft/`
- 启动器仅访问此目录及其子目录中的文件
- 不会修改 Minecraft 目录外的系统文件
- 所有文件操作都会记录日志以保证透明度

### 网络安全

- 所有到 Mojang/Microsoft 服务器的连接都使用 **HTTPS**
- 强制执行证书验证
- 启动器不会连接到未经授权的第三方服务器
- 网络请求会记录日志用于调试

## 用户安全最佳实践

### 一般指南

1. **保持启动器更新**到最新版本
2. **仅从官方来源下载**(GitHub releases)
3. **验证下载版本的完整性**
4. **审查启动器请求的权限**
5. 如果担心文件访问,**使用专用的 Minecraft 目录**

### 账户安全

1. **切勿在非官方启动器或网站中输入凭据**
2. **使用 Microsoft 的官方认证**流程
3. 不使用启动器时**退出登录**
4. 如果怀疑未经授权的使用,**撤销访问权限**

### 系统安全

1. **保持 macOS 更新**到最新版本
2. **使用 macOS 安全功能**(Gatekeeper、XProtect)
3. 如需要,**运行杀毒软件**
4. **监控系统日志**以发现异常活动

## 安全特性

### 已实现的安全措施

- ✅ **仅 HTTPS 连接**用于所有网络请求
- ✅ **SHA1 校验和验证**用于所有下载
- ✅ **安全令牌存储**使用 macOS 钥匙串
- ✅ **输入验证**用于用户提供的数据
- ✅ **全面日志记录**用于审计跟踪
- ✅ **代理支持**具有安全配置
- ✅ **不存储密码** - 仅基于令牌的认证

### 计划中的安全增强

- 🔄 **代码签名**用于 macOS 发布
- 🔄 **公证**用于 macOS 分发
- 🔄 **沙盒化**以增强隔离
- 🔄 **自动更新**具有签名验证
- 🔄 **增强日志记录**具有安全事件监控
- 🔄 **渗透测试**在稳定版本发布前
